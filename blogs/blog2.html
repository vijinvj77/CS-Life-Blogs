Java Session 2

On the second session, I learnt about casting an array datatype, String, Arrays, Relational and Logical operators in Java, Precedence order of Logical operators, Comments in Java, Errors and debugging.

Strings are used when we need to store more than one character. String values are always stored inside double quotes(““). The String concatenation is an important aspect in String. Addition of two or more strings is done by string concatenation. A “+” is used for this purpose.

Arrays- All the elements in an array can be only of one data type. i.e. if an array is declared as String, only string type data can be stored at each index and no other data type is compatible.

Declaring an array: datatype nameOfArray[];

Relational and Logical Operators-

Relational Operator: >,<,>=,<=,==,!=. These are the relational operators.

Logical Operator: &&(AND), ||(OR), !(NOT).
These are the logical operators. AND operation results in a true value only both of the conditions involved are true. OR operation results in a true value even when one of the condition involved is true. Precedence of the logical operators is

  1. NOT
  2. AND
  3. OR
.
Comments in java- It is a good practice to follow while writing programs to write comments. A “//” is used to write comments in java.

Errors and debugging- There are mainly two types of errors.
  • Compile-Time Errors
  • Run-Time Errors

Compile-time errors are syntax errors. Compile-time errors occurs when codes are written in ways in which the Java compiler does not recognise. The compiler will display errors to alert you about something that will not compile, and, therefore, cannot be run.

Run-time errors will only occur when the code actually running. These errors are the most difficult as they lead to unexpected program crashes and bugs in your code, both of which can be hard to track down and fix.

Debugging- Debugging is finding and removing errors or bugs in a code.
